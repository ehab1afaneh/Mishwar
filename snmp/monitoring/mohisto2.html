The monitors have also "memory", they are not just reporting what is currently going on. Monitors are able to keep records of data gathering about daily normal traffic. Statistics are counted for preset time interval set by the network manager. Statistics for a large number of intervals can be kept by the monitor, but the number of intervals depends on the monitor abilities.

The history control table is used to set configuration parameters for history collection. The network manager set the number of intervals, and the monitor records the closest number of intervals it could provide. When the total number of records is reached, the monitor warps around, and the older record is replaced by a new one. The manager sets the sample time interval from 1 second to 1 hour.

The data gathering begins after the operator has completed a valid entry in the control table. The History Control Table describes the database that contains the results of the actual history sample. This database will contain a separate set of history records for each History Control Table that point to an interface.
And separate data tables will be used for other medias. The first table index identifies which History Control Table configures these statistics. The second one identifies the sample interval.
