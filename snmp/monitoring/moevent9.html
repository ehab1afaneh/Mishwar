(1.3.6.1.2.1.16.9)


This group includes the Event Table and the Log Table. The Event Table define the notification that takes place when an event is triggered. One of the notification way is to write an entry in the Log Table.

  • The Event Table.
    Events are configured in the Event Table. An entry identifies an event and indicates an action (logging, sending a trap ).
    If a trap is required, the eventCommunity variable said where it has to be send, another variable knows when this event was last triggered.

  • Log Table.
    This table records each event that needs to be logged. It provides the event number, an index that distinguishes occurrences of the same event, the occurrence time and an implementation-specific event description.

  • Event, Alarms and Packet Matching.
    The alarm table sets up problem detection. A detected alarm points to an event that defines warning and action : if an alarm rising or falling Threshold is crossed, a corresponding event is defined in a variable. The variable points to an entry in the Event Table and it indicates what to do (nothing, write an entry in the log table, send a trap, or both log and trap).
    The packet matching can also produce an event. The entry in the Channel Table identifies the index of an event to be triggered when a packet match is found. This event tells what to do (log and / or trap).

    DESIGNING REPORTS : A lot of information are provided by monitors . And they offer a lot of different displays for the variables. Monitors offer some type of report design tools that make it easy to select variables and displays in any format.
