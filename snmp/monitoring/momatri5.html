Here, the aim of this group is to know who is talking to whom and how much. More precisely, what will be counted is the flows of traffic and errors from each source to each destination. In the matrix group, there is : a control table and a data table.

The data table is double ; the first is used to measure the flow of packets between source and destination, the second is identical, the difference is the order of indexation.

  • Matrix Control Table : the table identifies an interface to watch and identifies the upper limit to the number of source-destination pairs that can be tracked (When the table gets filled up, old entries have to be deleted).

  • Matrix Source-to-Destination Table (matrixSDTable) : contain a set of statistics that tracks the amount of traffic between two systems. This table records simple counts of packet, bytes and errors. Thus, it counts traffic from a source to the destination. The monitor will create a new entry in the table when it detects new flows. When space is missing the monitor delete entries (usually the older stored information).
    This matrix is indexed as follow :
    1. Interface being monitored.
    2. Source physical address
    3. Destination physical address.

  • Matrix Destination-to-Source Table (matrixDSTable) : is identical to the preceding table except that the order of the indices is
    1. Interface being monitored.
    2. Destination physical address.
    3. Source physical address.
    It is convenient for the order you would have a get-next request .
