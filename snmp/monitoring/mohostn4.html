(1.3.6.1.2.1.16.5)

This group allows to know the busiest station on the network and which is really busy. The hostTopN group provides frameworks to configure what we want the report to tell us. The operator can choose the criteria which will give him the hosts matching with the parameters. These parameters are the following ones :
  1. Incoming packets
  2. Outgoing packets
  3. Incoming octets
  4. Outgoing octets
  5. Outgoing errors
  6. Outgoing broadcast packets
  7. Outgoing multicast packet.
The top N (N is the number of stations you want in your result table) report is extracted then, written in the hostTopNControlTable .
This includes :
  • the maximum number of hosts you want to track.
  • one of the 7 preceding parameters statistics.
  • the time the monitor started.
  • The duration of the reporting interval.

    Two copies of the time interval are created. One of them is counted down (hostTopNTimeRemaining) until it reaches 0. Thus, it can not be set to 0. And the interval setting is copied into the hostTopNDuration variable.

    The duration variable is immovable. But the hostTopNTimeRemaining variable will decrease second by second until it reaches 0, then the sampling results are written in the Host Top N table.
    The result of the counts configured are stored in the HostTopN table. Hosts that have the largest charge of the selected parameters are listed (sorted descending). At each report, hosts are indexed by their Top N order.
