The Address Translation (at) Group (1.3.6.1.2.1.3)

The Address Translation (at) subtree was defined in MIB-I and consists of a single table that contains the network layer address to 'physical' address equivalences. Therefore, this table allows to transmit traffic to the nest-hop system that is along the current best path to the destination. The next-hop system may be a router or it may be the final destination.

The Address Translation Table

The Address Translation Table contains the address mappings (or equivalences). The addresses needed to be listed in the Address Translation Table are the ones that we absolutely need to know in order to transmit traffic to a next-hop system :
  • The Physical address (atPhysAddress) that is on directly connected subnetworks.
  • The Network address (atNetAddress), corresponding to the physical address, that is needed in order to route datagrams now (e.g. IP Address).
These address translation entries can be :
  • entered manually (telephone numbers...)
  • discovered automatically by a special protocol such as the Address Resolution Protocol (ARP)
  • calculated automatically.
Some interfaces do not use translation tables to determine address equivalences ; if all interfaces are of this type, then the Address Translation table is empty (has zero entry).

Deprecated Status of at in MIB-II

When MIB-II was written, the old at group was included without change, but was deprecated to conserve the compatibility between the two MIBs . It was decided that it was better to place separate address translation tables within the MIBs for each different Network Layer protocol.
